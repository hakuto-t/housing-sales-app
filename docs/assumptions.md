# 実装上の仮定と理由

## 採用した仮定

### 認証
- **仮定**: JWT + Cookie方式でシンプルな認証を実装
- **理由**: 初版では最小限のセキュリティで十分。OAuth/SSO等は将来拡張
- **制約**: パスワードリセット機能なし、セッション無効化はCookie削除のみ

### AI採点
- **仮定**: OpenAI互換APIを使用し、プロバイダを抽象化
- **理由**: 初版ではOpenAI API前提だが、将来のプロバイダ変更に備える
- **制約**: AI採点の精度はLLMの性能に依存。暫定スコアのフォールバックあり

### マルチテナント
- **仮定**: Company IDでデータを分離するが、初版は1社のみ
- **理由**: 将来の複数企業対応に備えたスキーマ設計
- **制約**: 企業の追加はDB直接操作が必要

### 用語データ
- **仮定**: 初期180語はseedで投入、CSVでの差し替えを想定
- **理由**: 実運用時に教育担当者がCSVで更新する運用を想定
- **制約**: 画像はURL指定のみ、ファイルアップロード機能なし

### 卒業検定
- **仮定**: 120問はDBにある問題から均等配分で出題
- **理由**: 問題プールが120問以上であることを前提とする
- **制約**: 問題数が不足するカテゴリ・タイプがある場合、配分が偏る可能性

### ランキング
- **仮定**: ランクはXP変動時にリアルタイム再計算
- **理由**: 少人数(数十名規模)を想定し、即時反映を優先
- **制約**: 数百名以上の場合はバッチ処理への切り替えが望ましい

## 未実装項目（将来拡張）

| 項目 | 理由 |
|------|------|
| 上司向けマネジメント機能 | 初版スコープ外 |
| 法改正追従システム | 初版スコープ外 |
| 高度なセキュリティ統制 | 社内利用前提で最小限 |
| 課金システム | 初版は無料運用 |
| 会社別カスタム問題 | マルチテナント拡張時に対応 |
| モバイル最適化 | PCが主対象、最低限の表示保証のみ |
| パスワードリセット | 初版ではDB直接操作で対応 |
| メール通知 | 初版スコープ外 |
| ファイルアップロード | URL指定で代替 |
| 学習レポートPDF出力 | 初版スコープ外 |
